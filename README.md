# Monte Test

Проект содержит экспериментальный конвейер моделирования. Ключевые материалы:

- [Архитектура решения](docs/architecture.md) — цели, ключевые компоненты, конвейер и контракты данных.
- `docs/` — дополнительная документация (будет пополняться по мере развития проекта).

## Структура кода

В репозитории представлен стартовый прототип ядра:

- `monte/` — исходный код Python-пакета.
  - `dsl/` — модель и парсер DSL.
  - `generator/` — генерация JSON-конфигурации запуска.
  - `execution/` — минимальное ядро исполнения и агрегации метрик.
  - `pipeline/` — оркестратор, связывающий парсер, генератор и исполнение.
  - `web/` — Flask-приложение с HTML-интерфейсом и REST API.
- `examples/` — примеры DSL-сценариев (`baseline.dsl`).
- `tests/` — модульные тесты.

CLI доступен через `python -m monte run <путь к DSL>` или установленный entry point `monte`.

## Веб-интерфейс

После установки зависимостей запустите локальный сервер:

```bash
pip install -e .
monte-web  # или: flask --app monte.web.app run --debug
```

Приложение откроет форму ввода DSL-сценария и покажет рассчитанные метрики, сырые результаты и JSON-конфигурацию. POST эндпоинт `/api/run` принимает JSON `{ "dsl": "..." }` и возвращает вычисленные данные, что позволяет интегрировать симулятор с внешними UI или скриптами.

## Быстрый старт
1. Изучите [архитектурный документ](docs/architecture.md), чтобы понять ключевые сущности и потоки данных.
2. Установите зависимости `pip install -e .` (опционально в виртуальном окружении).
3. Подготовьте описания сценариев в корпоративном репозитории DSL.
4. Запустите сценарий: `python -m monte run examples/baseline.dsl --json-output build/simulation.json` или через веб: `monte-web`.
5. Запланируйте разработку компонентов согласно разделу «План развития» документа по архитектуре.
