# Архитектура решения

## Цели
- Обеспечить единообразное описание экспериментов и сценариев с помощью DSL, пригодного для чтения человеком.
- Автоматизировать конвейер подготовки данных, запуска моделирования и агрегации результатов.
- Сохранять результаты в структурированном JSON-формате, пригодном для интеграции с внешними системами.
- Обеспечить расширяемость ядра за счёт модульной архитектуры и минимальной связности компонентов.

## Конвейер
1. **Подготовка входных данных** – загрузка исходных данных и конфигурации эксперимента из DSL-файлов.
2. **Парсинг DSL** – трансформация декларативного описания сценариев в промежуточное представление.
3. **Генерация JSON-конфигурации** – унификация параметров симуляции и сериализация их в стандартизированный JSON.
4. **Выполнение ядра** – запуск основных вычислений/моделирования на основе подготовленных параметров.
5. **Постобработка** – агрегация, нормализация и подготовка результатов для дальнейшего использования.
6. **Экспорт** – сохранение выходных данных в файлы или передачу в внешние сервисы.

## Структура JSON
- `metadata`: общая информация об экспериментах (версия схемы, дата запуска, инициатор).
- `inputs`: нормализованные входные данные и параметры симуляции.
- `pipeline`: описание этапов, которые были активированы для данного запуска.
- `results`: агрегированные результаты и ключевые метрики.
- `artifacts`: ссылки на дополнительные файлы или логи.

Каждый из разделов использует стабильные ключи и допускает расширения через поле `extensions`.

## DSL
- Основан на декларативном описании сценариев: сущности, параметры, зависимости.
- Поддерживает макросы и включения для переиспользования блоков конфигурации.
- Валидация происходит на этапе парсинга с использованием схемы, синхронизированной с JSON-структурой.
- DSL служит единственной точкой правды для описания экспериментов и автоматически транслируется в JSON.

## Ядро
- Включает движок выполнения, обрабатывающий очередь задач, сформированную после парсинга DSL.
- Реализует адаптеры для обращения к внешним вычислительным сервисам или локальным исполнителям.
- Поддерживает систему плагинов, позволяющих добавлять новые типы задач и обработчиков результатов.
- Собирает телеметрию для контроля качества и производительности.

## Расширяемость и интеграции
- Новые этапы конвейера добавляются через регистрационный механизм, не требующий изменения существующего кода.
- JSON-схема поддерживает версионирование для обратной совместимости.
- Предусмотрены точки интеграции с системами оркестрации и CI/CD.

## Тестирование и качество
- DSL и JSON проходят статическую валидацию согласно схемам.
- Для ядра предусмотрены модульные и интеграционные тесты, воспроизводящие ключевые сценарии.
- Конвейер покрывается end-to-end тестами, проверяющими взаимосвязь всех этапов.

## Дополнительные материалы
- Руководство пользователя описывает создание сценариев на DSL.
- Playbook оператора содержит инструкции по запуску и мониторингу конвейера.
